# Pédagogie

## Objectif

Définir comment le produit fait apprendre, pas seulement quoi enseigner.

`rumble-ai-practices` entraîne des réflexes professionnels. Sa pédagogie doit
favoriser le transfert vers le travail réel, pas la réussite artificielle à un
quiz.

## Principes pédagogiques

| Principe | Application produit |
| --- | --- |
| Situation avant définition | commencer par une décision concrète |
| Retrieval practice | faire réappliquer un réflexe dans plusieurs contextes |
| Feedback actionnable | expliquer risque, raison et prochaine action |
| Charge cognitive maîtrisée | un risque principal au niveau débutant |
| Métacognition | accepter « je ne peux pas conclure » quand le contexte manque |
| Espacement | proposer des retours par axe plutôt qu'une session unique |
| Transfert | relier chaque contenu à un usage professionnel réel |

## Formats d'exercices

| Format | Usage | Limite |
| --- | --- | --- |
| QCM contextualisé | MVP, validation rapide | peut masquer le raisonnement |
| Choix multiples | bonnes pratiques cumulatives | scoring plus complexe |
| Ordonnancement | priorité d'actions | demande feedback détaillé |
| Scénario à embranchements | arbitrage avancé | coût de production/revue |
| Fiche réflexe | consolidation | ne suffit pas sans pratique |
| Drill court | répétition ciblée | risque de mécanisation |

## Progression

### Débutant

- reconnaître sortie IA comme proposition ;
- protéger données et secrets ;
- vérifier une source citée ;
- comprendre les feedbacks par axe.

### Intermédiaire

- arbitrer outil/donnée/politique ;
- gérer RAG, documents et sources contradictoires ;
- identifier biais ou média synthétique problématique ;
- expliquer une réponse partielle.

### Avancé

- encadrer agents et tool calling ;
- documenter décisions à impact ;
- détecter prompt injection indirecte ;
- choisir escalade sécurité/DPO/juridique.

## Feedback comme moteur d'apprentissage

Le feedback doit faire plus que corriger. Il doit :

1. qualifier le choix ;
2. nommer le risque ;
3. expliquer pourquoi le risque existe ;
4. proposer l'action transférable ;
5. rappeler la source ou limite si nécessaire.

## Mesure de transfert

Indicateurs acceptables :

- progression par axe ;
- capacité à reconnaître une misconception dans plusieurs contextes ;
- baisse des choix `blocker` dans les reprises ;
- consultation de fiches de remédiation ;
- couverture compétences/risques du parcours.

Indicateurs refusés :

- vitesse de réponse ;
- classement nominatif ;
- note transmise au manager ;
- certification implicite.

## Anti-patterns pédagogiques

- Quiz de définitions IA.
- Vrai/faux sur des règles contextuelles.
- Prompt magique présenté comme compétence.
- Correction sans action.
- Gamification punitive.
- Scénarios trop longs au niveau débutant.
- Médias décoratifs qui augmentent la charge cognitive.

## Critères d'acceptation

- Chaque principe devient une règle produit ou éditoriale.
- Les risques de quiz superficiel sont traités.
- Le respect utilisateur est explicite.
- La progression reste compatible avec no-RH-scoring.
